    基础类型

    # 基础类型

    TS中的类型有一部分是JS当中就存在的，有一部分是新增的。

    1. 原本就有
      • 原始类型：number/string/boolean/null/undefined/symbol
      • 对象类型：object
    2. 新增：联合类型、自定义类型、接口、元组、字面量、枚举、void、any等

    # 数值类型

    双精度的64位浮点数，可以表示多种进制的数据

    let binary: number = 0b1010; // 二进制
let octal: number = 0o744;    // 八进制
let dec: number = 6;    // 十进制
let hex: number = 0xf00d;    // 十六进制

    # 字符串类型

    同样支持单引号''、双引号""、反引号``

    let name: string = "jack";
let gender: string = 'man';
let words: string = `hello`;

    # 布尔类型

    let flag: boolean = true;

    # Null和Undefined

    null表示空对象的引用，是一个只有一个值的特殊类型。

    undefined是用于初始化一个未设置值的变量。

    Null 和 Undefined 是其他任何类型（包括 void）的子类型，可以赋值给其它类型，如数字类型，此时，赋值后的类型会变成 null 或 undefined。而在TypeScript中启用严格的空校验（--strictNullChecks）特性，就可以使得null 和 undefined 只能被赋值给 void 或本身对应的类型，示例代码如下：

    // 启用 --strictNullChecks
let x: number;
x = 1; // 编译正确
x = undefined;    // 编译错误
x = null;    // 编译错误

    # 符号类型

    let symbol: symbol = Symbol();

    # 数组类型

    使用数组类型可以限制数组中元素的类型。具体有两种基本的写法：

    // 元素类型后面加上[]
let numbers: number[] = [1, 3, 5];
// 使用数组泛型
let strings: Array<string> = ['a', 'b', 'c'];

// 元素既有数值又有字符
let arr: (number | string)[] = [1, 'a', 2, 'b'];

    上述写法更推荐使用number[]。

    其中|在TS中叫做联合类型，是指由两个或多个其他类型组成的类型，表似可以是其中的任意一种。（后面会讲）

    # 函数类型

    函数类型实际上指的是：函数参数和返回值的类型。有两种指定方式：

    1. 分别指定参数、返回值的类型

      function add(a: number, b: number): number {
  return a + b
}

const add = (a: number, b: number): number => {
  return a + b
}

    2. 同时指定参数、返回值的类型（只适用于函数表达式+箭头函数的形式）

      const add: (a: number, b: number) => number = (a, b) => {
  return a + b
}

    函数参数的传递：JS中如果传递的实参少于或多于形参时，是不会报错的；但是TS会显示错误。

    TS中可选参数：在可传可不传的参数名称后面添加?

    不过要注意的是：可选参数只能出现在参数列表的最后，也就是说可选参数的后面不能出现必选参数。

    # 对象类型

    TS中的对象类型是在描述对象的结构（有什么类型属性和方法）

    let person: {name: string; age: number; speak(): void} = {
  name: 'Tom',
  age: 18,
  speak() {}
}

    说明：

    • 如果方法带有参数的话，类似函数参数，写成：speak(words: string): void即可。
    • 在一行指定对象类型时，各个属性之间用,或;分隔。
    • 如果通过换行指定对象类型时，可以省略各个属性之间的分隔符。
    • 同样，如果某个属性也是可选的，使用?即可。

    # 联合类型

    联合类型（Union Types）可以通过管道|将变量设置多种类型，赋值时可以根据设置的类型来赋值。

    格式：Type1 | Type2 | Type3

    例如：

    let val: string | number = 12
val = 'jack'

function disp(name: string | string[]) {...}

    # 自定义类型

    也称类型别名，为任意类型起别名。

    当同一类型（复杂）被多次使用时，可以通过类型别名，简化该类型的使用。

    type CustomArray = (number | string)[]
let arr: CustomArray = [1, 'a', 2, 'b']

    说明：

    • 自定义类型使用type关键字创建。
    • 类型别名可以是任意合法的变量名。

    # 接口

    接口是一系列抽象方法的声明，是一些方法特征的集合，这些方法都应该是抽象的，需要由具体的类去实现。

    当一个对象类型被多次使用，最好使用接口来描述，以达到复用的目的。

    interface Person {
  name: string
  age: number
  speak(): void
}

let person: Person {
  name: 'Tom',
  age: 18,
  speak() {}
}

    说明：

    • 接口使用interface创建。
    • 接口名可以是任意合法的变量名。
    • 接口不能编译成对应的JS代码，他只是TS的一部分。
    • 接口与类型别名的区别：
      • 接口只能为对象指定类型。
      • 类型别名可以为任意类型指定别名。

    当两个接口之间有共同的属性或方法，可以使用继承实现复用

    interface Animal {name: string}
interface Cat extends Animal {weight: string}

    说明：

    • 使用extends关键字实现了Cat继承Animal。
    • 继承之后，Cat也拥有了Animal的属性和方法。

    # 元组

    元组是另一种类型的数组，它确切地知道包含多少个元素，以及特定索引对应的类型。

    let position: [number, number] = [39, 116]

    应用场景：利用经纬度存储位置信息。

    # 字面量

    用于指定某个变量为某个值。

    let age: 18 = 18
const str = 'Hello TS'

    对比：

    let str1 = 'Hello TS'
const str2 = 'Hello TS'

    结果：

    1. 变量str1的类型为：string
    2. 常量str2的类型为：'Hello TS'

    # 枚举

    枚举定义了一组命名常量，描述一个值，该值是命名常量中的一个。

    枚举的功能类似字面量类型+联合类型，表示一组可选值。

    enum Day {Sunday, Monday, Tuesday, Wednesday, Thursday, Friday, Saturday}

console.log(Day.Sunday) // 0

    说明：

    • 枚举使用enum关键字创建。

    • 约定枚举名称和值以大写字母开头，值用,分隔。

    • 访问枚举成员，类似JS对象中的点语法。

    • 枚举的值默认是从0开始自增的数值，这时称为数值枚举。

    • 枚举的值还可以是字符串，每个成员都必须有初始值，这时称为字符串枚举。

      enum Direction {
  Up = 'UP',
  Down = 'DOWN',
  Left = 'LEFT',
  Right = 'RIGHT'
}

    • 枚举类型会编译为JS代码，因为枚举不仅作为类型，还提供值。可以使用常量枚举，避免编译时创建该对象。const enum ...

    一般情况下，推荐使用字面量类型+联合类型的方式，因为更直观、简洁、高效。

    # void

    用于标识返回值的类型，标识方法没有返回值。

    function add(): void {}

    # any

    声明为any的变量可以赋予任意类型的值。

    let x: any = 1;
x = 'I an who I am';
x = false

    说明：

    • 不推荐使用any，因为相当于不使用TS的类型特性。
    • 上面代码显示声明了any类型，隐式声明的情况：
      • 声明变量不提供类型也不提供默认值
      • 函数参数不声明类型

    # 类型的判断

    # 类型推论

    TS中带有类型推论机制，可以在没有明确给出类型的情况下推断出类型。

    let age = 18;
function add(a: number, b: number) {return a + b}

    这两种情况下，类型注解可以不写。

    # 类型断言

    有些情况下，变量的类型对于我们来说是很明确，但是TS编译器却并不清楚，此时，可以通过类型断言来告诉编译器变量的类型，断言有两种形式：

    1. let someValue: number | string = "this is a string"
let strLength: number = (someValue as string).length;

    2. let someValue: number | string = "this is a string"
let strLength: number = (<string>someValue).length;

    应用场景：

    <a href="http://..." id="link">链接</a>

    const linkDOM = document.getElementById('link') as HTMLAnchorElement

    说明： getElementById 方法返回值的类型是 HTMLElement，该类型只包含所有标签公共的属性或方法，不包含a 标签特有的 href 等属性。如果后续访问href属性，就会报错，通过类型断言使linkDOM的类型更具体。
