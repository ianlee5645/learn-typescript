    高级类型

    # 高级类型

    TS中的高级类型有很多，例如：class类，交叉类型，泛型，索引签名类型，映射类型。

    # class类

    # 介绍

    TS是面向对象的JS，支持面向对象的全部特性，比如类、接口等。

    TS全面支持ES2015中引入的class关键字，并为其添加了类型注解和其他语法。

    TS中的类包含以下模块（类的数据成员）：

    • 字段：字段是类里面声明的变量。字段表示对象的有关数据。
    • 构造函数： 类实例化时调用，可以为类的对象分配内存。
    • 方法： 方法为对象要执行的操作。
    class Car { 
    // 字段 
    engine: string;
    wheels = 4
 
    // 构造函数 
    constructor(engine: string) { 
        this.engine = engine 
    }  
 
    // 方法 
    disp(): void { 
        console.log("发动机为 : " + this.engine) 
    } 
}

const c = new Car('Engine 1')

    说明：

    • 根据TS中类型推论，实例对象c的类型是Car。
    • TS中的class，不仅提供了class的语法功能，也作为一种类型存在。
    • 方法的类型注解（参数和返回值）与函数用法相同。

    # 继承

    类实现继承有两种方式：

    1. extends（继承父类）

      class Animal {
  move(): void {console.log('move')}
}

class Cat extends Animal {
  miaow(): void {console.log('喵')}
}

const cat = new Cat()

    2. implements（实现接口）

      interface Singable {
  sing(): void
}

class Singer implements Singable {
  sing(): void {
     console.log('lalala')
  }
}

      实现接口意味着类中要提供接口中指定的所有方法和属性。

    # 类成员的访问控制修饰符

    TS中，可以使用访问控制符来保护对类、变量、方法和构造方法的访问。包括三类：

    • public:：表示公有的、公开的，公有成员可以被任何地方访问，默认可见性。

      class Animal {
  public move() {
    console.log('move')
  }
}

    • protected：表示受保护的，仅对其声明所在类和子类中（非实例对象）可见。

      class Animal {
  protected move(): void {console.log('move')}
}

class Cat extends Animal {
  miaow(): void {
    console.log('喵')
    this.move()  
  }
}

    • private：表示私有的，只在当前类中可见，对实例对象以及子类也是不可见的。

      class Encapsulate { 
   str1: string = "hello" 
   private str2: string = "world" 
}
 
var obj = new Encapsulate() 
console.log(obj.str1)     // 可访问 
console.log(obj.str2)   // 编译错误， str2 是私有的

    额外的修饰符：readonly

    readonly：表示只读，用来防止在构造函数之外对属性进行赋值，只能用来修饰属性。

    # 交叉类型

    # 介绍

    功能类似于接口继承，用于组合多个类型为一个类型（常用于接口），用&实现。

    interface Person {name: string}
interface Contact {phone: string}
type PersonDetail = Person & Contact
le obj: PersonDetail = {
  name: 'Tom',
  phone: '1818181818'
}

//相当于
type PersonDetail = {name: string; phone: string}

    # 特点

    交叉类型（&）与接口继承（extends）都可以实现对象类型的组合。不同的是对于同名属性的合并处理：

    接口继承（extends）：

    interface A {
    fn: (value: number) => string
}

interface B extends A {
    fn: (value: string) => string
}

    出现错误：

    交叉类型（&）：

    interface A {
    fn: (value: number) => string
}

interface B {
    fn: (value: string) => string
}

type C = A & B

    不出现错误：

    可以理解为：

    fn: (value: string | number) => string

    # 泛型

    # 介绍

    如果对于一个函数，我们的目的是传入什么数据就返回什么数据，也就是说参数和返回值的类型要一致。这样的话，很容易想到any：

    function fn(value: any): any {return value}

    但是使用any的话，会丢失TS的类型保护。所以需要借助泛型。

    泛型能够在保证类型安全的前提下，让函数，接口等可以与多种类型一起工作，从而实现复用，常用于函数，接口，类中。

    格式：

    function fn<Type>(value: Type): Type {return value}

    说明：

    • 在函数名称后面添加<>，然后在其中写入类型变量。
    • 类型变量 Type，是一种特殊类型的变量，它处理类型而不是值。
    • 类型变量相当于一个类型容器，能够捕获用户提供的类型。
    • Type可以是任意合法的变量名称。

    调用：

    const num = fn<number>(10)
const str = fn<string>('a')
// 简化调用
const num = fn(10)

    说明：

    • 在函数名称后面添加<>，然后在其中指定类型，之后类型变量Type就能捕获到类型。
    • 如果省略<>，TS内部能根据实参推断出Type的类型。

    # 泛型约束

    因为类型变量Type可以代表多个类型，这会导致意想不到的错误：

    function fn<Type>(value: Type): Type {
  console.log(value.length)
  return value
}

    我们知道原始类型的数据是没有length属性的。

    解决：给泛型添加约束，收缩类型：

    // 1.指定具体类型
function fn<Type>(value: Type[]): Type[] {
  console.log(value.length)
  return value
}
// 2.继承接口
interface ILength {length: number}
function fn<Type extends ILength>(value: Type): Type {
  console.log(value.length)
  return value
}

    特别注意：泛型的类型变量可以有多个，并且类型变量之间可以约束：

    function fn<Type, Key extends keyof Type>(obj: Type, key: Key) {
  return obj[key]
}
let person = {name: 'Tom', age: 18}
fn(person, 'name')

    说明：

    keyof是个关键字，接收对象类型，生成键名的联合类型。本例为："name" | "age"。

    # 泛型接口

    接口可以配合泛型来使用，增强灵活性。

    interface IdFunc<Type> {
  id: (value: Type) => Type
  ids: () => Type[]
}

let obj: IdFunc<number> = {
  id(value) {return value},
  ids() {return [1,2,3]}
}

    # 泛型类

    类也能配合泛型来使用。

    class Cal<Type> {
  value: Type
  add: (x: Type, y: Type) => Type 
}
const myCal = new Cal<number>()
myCal.value = 10

    # 索引签名类型

    在定义接口的时候，我们可以很明确地指定对象的结构，但是有时候无法确定对象中会出现哪些属性（或者对象中可以出现任意属性），这时需要借助索引标签类型。

    interface MyObj {
  [key: string]: number
}

    说明：

    • 上面的接口约束了对象的键只能是字符串类型，值是数值类型。
    • 通过这种方式，只要满足要求，对象中 可以出现任意多的属性。
    • key只是占位符，可以是任意合法的变量名称。

    # 索引查询类型

    可以查询属性的类型：

    type Props = {a: number, b: string, c: boolean}
type TypeA = Props['a']  // number
type TypeA = Props['a' | 'b']  // number | string
type TypeA = Props[keyof Props]  // number | string | boolean

    # 映射类型

    基于旧类型创建出新类型（对象类型），减少重复：

    // 普通写法
type PropKeys = 'x' | 'y' | 'z'
type TypeA = {x: number, y: number, z: number}
// 映射类型写法
type PropKeys = 'x' | 'y' | 'z'
type TypeA = {[k in PropKeys]: number}

    说明：

    • k in PropKeys表示 k可以是 PropKeys 联合类型中的任意一个。
    • 映射类型只能在类型别名中使用，不能在接口中使用
